计算模式

MLIP+VASP 通常需要在 server / jobflow-remote 集群上运行，并且本地或服务器一侧已经配置好 jobflow worker。

本机模式只支持 MLIP。要做 VASP 必须选择 server 并配置 jobflow-remote。

SLURM 资源（仅 server 模式）
分区 CPU/GRES 占用与队列汇总（sinfo / squeue），不含节点逐台列表。
结构与名称
从 sinfo 拉取分区名并填入 MLIP / VASP / Si 探针的下拉菜单（不含节点）。

srun -p；需与上面 device 一致用于 MLIP 探针。

可选：验证环境与 Si 静态能；探针 device 与上方一致。不检测时也可直接在下拉里选 worker 提交。

server 模式下会列出当前 project 中的全部 worker；点击「检测」后可看到 import / Si 状态。

MLIP 作业资源（仅 server 模式）

jobflow-remote 项目名（默认 JFREMOTE_PROJECT / std）。改动后请重新「检测 MLIP worker」。

将合并到 MLIP 作业 scheduler_kwargs。常见键如 qos、mem、account、time、nodelist。

VASP 资源（仅 MLIP+VASP）
从 ~/.jfremote/<project>.yaml 读取 workers。
选中后自动写入下方 vasp_pre_run（按选择顺序拼接 module load …）。

可手动编辑；若上方 module 选择发生变化，将整段覆盖为 module load … && …。

VASP 输入（INCAR / KPOINTS / POTCAR；仅 MLIP+VASP）

决定提交多少套界面 VASP 弛豫。

使用 pymatgen DictSet 支持的键（reciprocal_density、grid_density 等），不要直接写 kpoints 网格列表。
